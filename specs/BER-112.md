# BER-112: Detailansicht — Navigation zum vorherigen/nächsten Beleg

> **Pre-Flight:** Lies `CLAUDE.md`, dann `docs/UEBERGABE.md`.

## Kontext

Beim Durcharbeiten des 2024-Stapels (59 Belege) ist der Weg
„Detail → zurück zur Liste → nächster Beleg" zu umständlich.

## Anforderung

In der Detailansicht sollen Vorgänger und Nachfolger direkt aufrufbar sein —
per Pfeil hoch/runter, ohne Umweg über die Liste.

## Akzeptanzkriterien

- [x] Pfeil-Buttons (hoch = vorheriger, runter = nächster Beleg) in der Detailansicht
- [x] Tastatursteuerung: `ArrowUp` / `ArrowDown` navigieren direkt
- [x] Reihenfolge identisch zur Belegliste
- [x] An den Rändern deaktiviert (kein Umlauf)
- [x] Keine Tastatur-Übernahme, während ein Eingabefeld fokussiert ist

## Technische Notizen

Die Liste sortiert nach `created_at DESC`. Für die Navigation wird der
**zusammengesetzte Schlüssel `(created_at, id)`** per Row-Value-Vergleich benutzt —
bei Batch-Importen können mehrere Belege denselben `created_at` haben, wodurch die
Reihenfolge sonst nicht eindeutig wäre. Die Listensortierung wurde entsprechend auf
`created_at DESC, id DESC` stabilisiert, damit Liste und Navigation exakt
übereinstimmen.

Tastatur-Handling in einer Client-Komponente (`beleg-navigation.tsx`); Events werden
ignoriert, wenn `INPUT`/`TEXTAREA`/`SELECT`/contentEditable fokussiert ist oder
Modifier-Tasten gedrückt sind — sonst würde das Freigabe-Formular gestört.

**Verifiziert** gegen die Datenbank: für Listenposition 5 (`01-2026-0055`) liefert die
Vorgänger-Query `0056` (Position 4) und die Nachfolger-Query `0054` (Position 6);
erster Beleg hat 0 Vorgänger, letzter 0 Nachfolger.
