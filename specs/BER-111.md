# BER-111: Import-Watcher — überlappende Scans verhindern

> **Pre-Flight:** Lies `CLAUDE.md`, dann `docs/UEBERGABE.md`.

## Kontext

Beim Import des 2024-Stapels (51 PDFs) schlugen 11 Dateien fehl: 8× HTTP 500
(„Try spacing your requests out" — KI-Rate-Limit) und 3× HTTP 409
(`duplicate key … belege_beleg_nr_key` — zwei Importe berechneten gleichzeitig
dieselbe Belegnummer). Ein sequentieller Retry derselben 11 Dateien lief
**11/11 fehlerfrei** durch.

## Anforderung

Der Watcher darf Importe nicht parallel ausführen.

## Akzeptanzkriterien

- [x] Ein `setInterval`-Tick wird übersprungen, solange der vorherige Scan läuft
- [x] Das bestehende Pacing zwischen Einzelimporten bleibt erhalten
- [x] Verhalten sonst unverändert (Archivierung, Fehler-Ordner, Duplikaterkennung)

## Technische Notizen

Ursache in `scripts/beleg-import/beleg-import.mjs`, `cmdWatch`: `setInterval(scan, 5000)`
startet den nächsten Scan unabhängig davon, ob der vorherige noch läuft. `scan()` ist
async und awaited jeden Import; bei langsamen Importen (OCR + KI) überlappen sich
mehrere Scans. Das `busy`-Set verhindert nur die Doppelverarbeitung *derselben* Datei,
nicht Parallelität insgesamt. Fix: Lock-Flag um den Tick.

Die härtere Absicherung — Retry auf `23505` bei der Belegnummern-Vergabe — bleibt als
Folge-Issue offen; sie wäre nötig, falls je mehrere Import-Prozesse gleichzeitig laufen.
